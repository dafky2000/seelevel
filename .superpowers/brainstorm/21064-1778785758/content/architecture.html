Extension Architecture

ViewPoint Analytics — Chrome MV3, native Side Panel, intercept-only, no storage

🌐 viewpoint.ca page
Google Maps + Listings
The live page. Makes XHR/fetch calls to /api/v2/listing/search. Results rendered in the listing sidebar and map pins.
window.fetch (patched)
Content script injects into MAIN world and wraps fetch. Intercepts search API responses before the page processes them — zero visual side-effects.
TS MAIN world
⚙️ content script
fetch-interceptor.ts
Patches window.fetch in MAIN world. Detects listing search API calls, parses response with reused parseSearchResponse() logic. Posts ListingRow batches to ISOLATED world via custom event.
TS
relay.ts
Runs in ISOLATED world. Listens for intercepted batches, detects active status filter from URL/DOM, forwards to side panel via chrome.runtime.sendMessage.
TS ISOLATED
geofence-overlay.ts
Injects "Draw Zone" button onto the viewpoint map toolbar. On click, activates Leaflet canvas overlay for polygon drawing (Leaflet + Geoman, bundled). Sends completed polygon coords to relay.
TS Leaflet + Geoman
📊 side panel
App.tsx (Preact root)
Receives listing batches via chrome.runtime.onMessage. Holds all session listings in a useRef accumulator. Closing the panel = data gone.
Preact @dano/styles
Analytics engine
Pure functions: filter by geofence polygon (point-in-polygon), filter by time window, compute avg/median/stddev/count/DOM. Runs in-memory on every update.
TS
Charts + Export
Time-series charts (lightweight canvas lib — uPlot or Chart.js). Metric tabs: Avg Price, Volume, DOM, $/sqft. Export button downloads aggregated buckets as CSV — no raw listing data.
